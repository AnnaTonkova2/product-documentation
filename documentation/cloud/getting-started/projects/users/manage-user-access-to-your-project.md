#### Projects are goal-oriented containers for the Cloud resources within which you can create instances, volumes, networks, and other resources you need.  

<img src="https://support.gcore.com/hc/article_attachments/360008320277/1_____.png" alt="1_____.png">
-----------------------------------------------------------------------------------------------------


How to create a project 
------------------------

Every user already has a default project created automatically. This project cannot be deleted.  To create a new project, click the Create project button in the Projects tab.  

<img src="https://support.gcore.com/hc/article_attachments/360008403518/2_____.png" alt="2_____.png">Enter a name and description of the project and click Create project.  

**Important!** The Name and Description field lengths can be from 3 to 63 characters. Lowercase Latin letters, numbers, and hyphens are allowed. Name and Description **should not** end with a hyphen.  

<img src="https://support.gcore.com/hc/article_attachments/360008320297/3_____.png" alt="3_____.png" width="505" height="288">

The amount of your project doesn't affect the service price directly. Payment is calculated based on the resources used in the projects.

You can Open each project, go to its Settings or Delete it. To do this, click the icon <img src="https://support.gcore.com/hc/article_attachments/360008319897/_________.png" alt="_________.png" width="31" height="25">  next to the project you want to manage.

<img src="https://support.gcore.com/hc/article_attachments/360008320317/4_____.png" alt="4_____.png">By clicking the Open button you will be redirected to the resources management panel. 

Project settings
----------------

                         

In the project Settings, you can edit the project name and description, and configure access rights.

<img src="https://support.gcore.com/hc/article_attachments/360019282717/30.png" alt="30.png">

Here you can change the project name and description. The data will be changed after saving.

<img src="https://support.gcore.com/hc/article_attachments/360019282677/29.png" alt="29.png">

The Access tab displays all the users who have access to the selected project.  

<img src="https://support.gcore.com/hc/article_attachments/360019282697/28.png" alt="28.png">

### Roles of users in a project

Certain roles are assigned to all users in projects. Rights to manage settings, resources, and other users are set according to assigned roles. 

All available roles, their descriptions, and the rights assigned to them are listed below.    

| \nRole \n                                                 | \nDescription \n                                                                                                                                                                                                                                              | \nRights \n                                                                                               |
|-----------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------|
| \nAccount owner (administrator of the default project) \n | \nThe user has full access to all the Gcore Cloud functionality (if not restricted within the quotas) for all projects. A user can create resources, projects, and manage users for any project. \n                                                           | \nAll Gcore Cloud functionality for a client. \n                                                          |
| \nAdministrator \n                                        | \nThe user has full access to all the Gcore Cloud functionality (if not restricted within the quotas) only within their project and does not have access to other projects. A user can create resources, projects, and manage users within their projects. \n | \nAll Gcore Cloud functionality for a client within their projects. \n                                    |
| \nUser \n                                                 | \nThe user has access to a specific project. A user can manage resources, but can't create projects/manage users. \n                                                                                                                                          | \nResources management within a project. \n                                                               |
| \nObserver \n                                             | \nThe user has access to a specific project, except a default one. Read-only access. A user can only see resources without the ability to manage resources, projects, or users. \n                                                                            | \nRead-only access to a project. \n                                                                       |
| \nInternal Network Only User\n                            | \nThe user can create resources only in private networks and cannot in public ones\n                                                                                                                                                                          | \nResources management within a project, restrictions on the creation of resources in a public network.\n |


### How to edit user's role

If you have enough rights, you can regulate user rights. Click Edit to go to the user settings. 

<img src="https://support.gcore.com/hc/article_attachments/360019348538/27.png" alt="27.png">

You can change the user role in the column Role. There are available the following roles: User, Observer and Administrator. Save and confirm changes.  

<img src="https://support.gcore.com/hc/article_attachments/360019348558/26.png" alt="26.png">

### How to deny access to a project

To deny access to a project, click the Delete role button in the user settings. 

<img src="https://support.gcore.com/hc/article_attachments/360019348498/25.png" alt="25.png">

 After confirming the deletion, a user will not be allowed to access the project. 

In the Access section of project settings, the user will be displayed with the Non-assigned role. 

<img src="https://support.gcore.com/hc/article_attachments/360019348518/24.png" alt="24.png">

### How to restore access to a project for a user

To invite a user to the project again, click Edit.

<img src="https://support.gcore.com/hc/article_attachments/360019348478/23.png" alt="23.png">

Change the user role to the required one and save the settings. 

<img src="https://support.gcore.com/hc/article_attachments/360019348458/22.png" alt="22.png">

Confirm the role reassigning.

<img src="https://support.gcore.com/hc/article_attachments/360019282657/21.png" alt="21.png">

After the changes, the user will again have access to a project via the personal account. 

### How to invite a new user to a project

If you are The Owner or Administrator of the project, you can add a new user. Click the Invite Users button. 

<img src="https://support.gcore.com/hc/article_attachments/360019348438/20.png" alt="20.png">

In a pop-up window, specify a user's email address and a role in the project. 

<img src="https://support.gcore.com/hc/article_attachments/360019282637/19.png" alt="19.png">

Click the Add another user button to add more users and click Send Invite.

<img src="https://support.gcore.com/hc/article_attachments/360019348418/18.png" alt="18.png">

**Important!** You can invite only users who were not previously registered in the Gcore system. 

Invitations from [**support@gcore.com**](mailto:support@gcorelabs.com) will be sent to the addresses specified in the invitation.   

**The invitation should be accepted by an invited user within 24 hours.** 

<img src="https://support.gcore.com/hc/article_attachments/360019348598/17.png" alt="17.png">

After accepting an invitation, a user will be redirected to the password creation page. 

<img src="https://support.gcore.com/hc/article_attachments/360019282777/16.png" alt="16.png">

The password should consist of at least 8 characters and contain an uppercase letter, lowercase letter, number, and at least one special character (for example, @,#,$,%,^). 

To save the password click Join. In the pop-up notification, click **Log in**. 

Then enter a created password and click Log in. The user's email address has already been entered automatically.  

How to delete a project
-----------------------

If your [user's role](#roles-of-users-in-a-project) is Account owner or Administrator, you can delete projects. To do it, select Delete in the menu and confirm the deletion. 

<img src="https://support.gcore.com/hc/article_attachments/360008320337/5_____.png" alt="5_____.png">

! The default project cannot be deleted.
